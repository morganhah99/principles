Principles of Object Oriented Programming

Inheritance

Inheritance is the capability of one class to derive or inherit the properties from another class. The benefits of inheritance are:

  • It represents real-world relationships well
  • It provides reusability of a code and allows us to add more features to a class without modifying it.
  • It is transitive in nature, which means that if class B inherits from another class A, then all the subclasses of B would automatically inherit from class A

Polymorphism

Polymorphism simply means to have many forms. In programming, polymorphism means the same function name being used for different types. A simple example would be, print(len("length")) and print(len([10,20,30])) with the output being 6 and 3.

Abstraction

An abstract class can be considered as a blueprint for other classes. It allows you to create a set of methods that must be created within any child classes built from the abstract class.
By defining an abstract base class, you can define a common Application Program Interface (API) for a set of subclasses. The capability is especially useful where a third-party is going to
provide implementations, such as with plugins, but can also help you with a large code-base where keeping all classes in your mind is difficult or not possible.

Encapsulation

Encapsulation is one of the fundamentals concepts in Object Oriented Programming. It describes the idea of wrapping data and the methods that work on data within on unit. This puts
restrictions on accessing variables and methods directly and can prevent the accidental modification of data. To prevent accidental change, an object's variable can only be changed
by an object's method. Those types of variables are known as private variables and are defined with a double underscore as a prefix to a name
